# Awesome MCP Korea Analysis

분석 대상: `darjeeling/awesome-mcp-korea`

저장소: <https://github.com/darjeeling/awesome-mcp-korea>

## 요약

이 저장소는 한국 시장과 한국 서비스에 특화된 MCP 서버를 카테고리별로 모아 둔 큐레이션 저장소다. 핵심 가치는 단순한 MCP 소개가 아니라, `한국형 에이전트에 어떤 외부 능력을 붙일 수 있는가`를 빠르게 훑어볼 수 있는 출발 인덱스를 제공한다는 점에 있다.

현재 공개 GitHub 페이지 기준으로 이 저장소는 `Public`, `Star 36`, `Fork 3`, `No releases published` 상태이며, 목록 자체는 `CC0 1.0 Universal` 라이선스로 공개돼 있다.

## 왜 GovRadar에 두는가

이 저장소는 개별 MCP 서버 하나가 아니라 `한국형 공공/산업 데이터 접근 생태계`를 묶어서 보여주는 메타 인덱스에 가깝다. 따라서 현재 워크스페이스에서 가장 자연스러운 1차 적재 위치는 `GovRadar`다.

이유는 세 가지다.

- `GovRadar`는 이미 공공기관 지원사업, 정책 브리핑, 공공 API 품질 신호를 함께 수집하고 있다.
- `awesome-mcp-korea`에는 법률, 공공데이터, 부동산, 지도, 날씨, 관광처럼 한국 공공/준공공 데이터 활용과 직결되는 카테고리가 많다.
- 이 저장소는 정책 뉴스 그 자체보다 `한국형 AI 도구 생태계가 어디서 실제 연결점을 만들고 있는가`를 보여주는 신호원으로 더 적합하다.

## 공식 저장소 기준 핵심 포인트

README 기준으로 이 저장소는 다음 조건을 충족하는 프로젝트만 받는다.

- 공개 접근 가능
- 한국 생태계를 명확히 지원
- MCP 명세 구현
- 기본 문서 포함
- 마케팅 전용 저장소 제외

또한 저장소는 명시적으로 `보안, 안전성, 신뢰성`을 보증하지 않으며, 실제 도입 전에는 사용자가 개별 프로젝트의 인증 방식, 권한 범위, 키 처리, 유지보수 상태를 직접 검토해야 한다고 적고 있다.

## 카테고리별 사업적 해석

### GovRadar 직접 관련

- `Legal & Government`
  한국 법령, 판례, 공공 서비스용 MCP. 공공기관 업무지원, 법령 검색 자동화, 행정 QA 보조에 직접 연결된다.
- `Public Data`
  서울시 데이터, data.go.kr, 도서관정보나루, 식약처, 지하철, KOSIS 등. GovRadar의 공공 데이터 활용 축과 가장 직접적으로 이어진다.
- `Tourism & Travel`
  한국관광공사 TourAPI 계열. 지역 공공 서비스와 로컬 방문자 서비스를 다루기 때문에 지자체/공공기관 활용 시나리오와 겹친다.
- `Weather`
  기상청 OpenAPI 계열. 재난, 교통, 야외행사, 관광, 현장작업 지원 같은 공공 서비스 시나리오로 연결된다.

### 다른 Radar로 넘길 후보

- `PolicyRadar`
  법률/정부 카테고리, 국가법령정보 Open API 계열 MCP를 정책·규제 자동화 관점에서 추적하기 적합하다.
- `PropertyRadar`
  실거래가, 청약홈, 온비드, 부동산 투자분석 MCP는 별도 부동산 워크플로우로 확장 가치가 높다.
- `TrustRadar`
  이 목록은 보안 면책을 명시하므로, 실제 도입 전 공급망 위험, 자격증명 관리, 권한 범위 검토를 보안 관점에서 다룰 필요가 있다.
- `PaperworkRadar`
  법률/공공 서비스/문서 파싱/HWP-PDF 파싱 계열 MCP는 행정 문서 처리 자동화와 궁합이 좋다.

## 이 워크스페이스에서의 실전 활용 방식

이 저장소는 `바로 도입할 MCP 서버`를 뜻하지 않는다. 현재 워크스페이스에서의 올바른 쓰임새는 아래와 같다.

1. `카테고리 레벨 레이더`
   개별 도구보다 먼저 어떤 분야가 빠르게 형성되는지 본다.
2. `후보 발굴 인덱스`
   법률, 부동산, 공공데이터처럼 우선순위 영역별 후보 3~5개만 추린다.
3. `실사 대상 큐`
   인증 방식, API 키, 유지보수 상태, 라이선스, 최근 커밋, 실제 API 의존성을 별도 검토한다.

즉, 이 저장소는 운영 데이터 자체보다 `후보군 축약`에 가치가 있다.

## GovRadar에 추가한 해석

이번 작업에서는 이 저장소를 단순 GitHub 커밋 피드가 아니라 `한국형 MCP 생태계 신호`로 읽을 수 있게 정리했다.

- `govsupport.yaml`에 이미 등록된 `Awesome MCP Korea` 소스를 유지
- `McpKoreaEcosystem` 엔티티를 추가해 MCP, 공공 API, 한국형 AI 도구화 관련 신호를 별도 태깅
- README/AGENTS에서 이 저장소를 `GovRadar`의 공공 API/MCP 추적 축으로 명시

## 권장 후속 작업

우선순위는 아래 순서가 적절하다.

1. `GovRadar`
   `awesome-mcp-korea`와 `K-GAPI`를 묶어 한국 공공 API/MCP 생태계 섹션을 리포트에 분리
2. `PolicyRadar`
   국가법령정보 기반 MCP, 정부 법령 검색 MCP를 정책 자동화 시그널로 태깅
3. `PropertyRadar`
   실거래가/청약/온비드 계열 MCP를 투자/실거래 워크플로우로 확장
4. `TrustRadar`
   MCP 서버 도입 전 체크리스트를 공급망/권한/비밀관리 관점으로 추가
