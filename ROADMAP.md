# GovRadar 품질 개선 로드맵 (Quality Improvement Roadmap)

> 정부 정책 및 공공 데이터 수집 및 분석 파이프라인 품질 고도화 계획

## 현재 상태 (Current Status)

| 지표 | 현재 값 | 목표 |
|------|---------|------|
| 소스 수 | 26개 | 35개+ |
| 매칭률 | 57% | 80%+ |
| 테스트 수 | 3개 | 15개+ |
| P0 이슈 | 1개 | 0개 |

**상태 요약**: 26개 소스를 운영 중이나 config 구조에 혼재된 필드가 있어 정리가 필요합니다. 매칭률 57%로 키워드 개선도 시급합니다.

## Phase 0: 긴급 수정 (P0 - 긴급)

### 0.1 Config 구조 정리

**문제**: config 파일에 혼재된 필드가 있어 파싱 오류 및 유지보수 어려움 발생

**목표**: config 구조 표준화 및 불필요한 필드 정리

**작업 내용**:
- [ ] 현재 config 분석
  - `config/categories/gov.yaml` 필드 구조 검토
  - 중복/불필요한 필드 식별
  - 타 레이더 config와 비교 분석
- [ ] 표준 config 구조 적용
  - radar-core 표준 스키마 준수
  - 필수 필드 검증 (name, url, type, tier)
  - 선택 필드 정리 (keywords, schedule 등)
- [ ] 소스별 필드 정규화
  - RSS 소스: feed_url, category 필드 통일
  - API 소스: endpoint, auth 필드 표준화
  - 웹 파싱 소스: selector, pagination 필드 정리
- [ ] 유효성 검사 추가
  - config 로딩 시 스키마 검증
  - 누락 필드 경고 로그
  - 잘못된 URL 형식 감지

**검증 기준**:
- config 파싱 오류 0건
- 모든 소스 필드 표준화 완료
- config 유효성 검사 통과

## Phase 1: 매칭률 및 테스트 개선 (P1 - 높음)

### 1.1 매칭률 개선

**목표**: 매칭률을 57%에서 80% 이상으로 향상

**작업 내용**:
- [ ] 정책 분야별 키워드 확장
  - 경제 정책: 금리, 물가, 고용, 부동산 정책 등
  - 사회 정책: 복지, 교육, 의료, 환경 등
  - 외교/안보: 국제 관계, 북한, 국방 등
  - 과학/기술: R&D, 디지털, AI, 우주 등
- [ ] 정부 기관별 키워드 추가
  - 중앙부처 18개 키워드
  - 청와대/대통령실 관련 키워드
  - 국회/정당 관련 키워드
  - 헌법기관 키워드
- [ ] 정책 용어 사전 구축
  - 정책 약어 풀네임 매핑 (K-뉴딜, 탄소중립 등)
  - 법안명 키워드 (특별법, 개정안 등)
  - 예산/재정 용어 (추경, 본예산 등)
- [ ] 매칭 로직 고도화
  - 복합 키워드 매칭 (정부 + 정책 동시 출현)
  - 정책 발표 패턴 인식 (발표, 시행, 개정 등)

**검증 기준**:
- 매칭률 80% 이상
- 주요 정책 뉴스 누락률 5% 미만
- 오탐률 10% 미만

### 1.2 테스트 커버리지 확대

**목표**: 테스트 수를 3개에서 15개 이상으로 확대

**작업 내용**:
- [ ] `test_collector.py`: 수집기 단위 테스트 5개 추가
  - RSS 파싱 테스트
  - 정부 보도자료 구조화 테스트
  - 에러 핸들링 테스트
  - 중복 감지 테스트
  - 정책 분야 분류 테스트
- [ ] `test_analyzer.py`: 분석기 테스트 4개 추가
  - 기관 엔터티 추출 테스트
  - 정책 분야 분류 테스트
  - 트렌드 점수 계산 테스트
  - 키워드 매칭 테스트
- [ ] `test_storage.py`: 저장소 테스트 3개 추가
  - DuckDB 쓰기/읽기 테스트
  - 스냅샷 생성 테스트
  - 데이터 무결성 테스트
- [ ] `test_reporter.py`: 리포터 테스트 3개 추가
  - HTML 생성 테스트
  - JSON 요약 테스트
  - 정책 분야별 차트 테스트
- [ ] `test_config_loader.py`: config 테스트 추가
  - 스키마 유효성 테스트
  - 필수 필드 검증 테스트

**검증 기준**:
- 테스트 커버리지 60% 이상
- 모든 테스트 통과
- CI 파이프라인 자동 실행

## Phase 2: 소스 확장 (P2 - 중간)

### 2.1 지방자치단체 소스 추가

**목표**: 지방정부 정책 및 공고 수집 기능 추가

**작업 내용**:
- [ ] 광역시도 소스 추가
  - 서울시 정책 뉴스/보도자료
  - 경기도 정책 뉴스/보도자료
  - 부산/인천/대구 등 광역시
  - 각 시도 공공 데이터 포털
- [ ] 기초자치단체 소스 추가
  - 주요 시군구 보도자료
  - 지역 정책 RSS 피드
  - 지방의회 회의록/공고
- [ ] 지방 정책 분류 체계
  - 지역별 태깅 시스템
  - 지역 정책 vs 중앙 정책 구분
  - 지역 현안 키워드 추가

**검증 기준**:
- 지자체 소스 15개 이상 추가
- 지역 정책 매칭률 70% 이상

### 2.2 국회/입법 소스 추가

**목표**: 국회 입법 활동 및 법안 추적 기능 추가

**작업 내용**:
- [ ] 국회 소스 추가
  - 국회 의안정보시스템 연동
  - 국회 보도자료 RSS
  - 국회 본회의/상임위 일정
- [ ] 법안 추적 기능
  - 발의 → 심사 → 본회의 단계 추적
  - 주요 법안 자동 태깅
  - 법안 통과/폐기 알림
- [ ] 정당 동향 추적
  - 주요 정당 보도자료
  - 정책 공약 발표
  - 여야 정책 비교

### 2.3 공공 데이터 API 연동

**목표**: 공공 데이터 포털 API 연동으로 정량 데이터 확보

**작업 내용**:
- [ ] 공공 데이터 포털 API 연동
  - data.go.kr 주요 API
  - 통계청 KOSIS API
  - 한국은행 경제통계 API
- [ ] 정책 효과 지표 수집
  - 고용/실업률 통계
  - 물가/경제 지표
  - 복지/사회 지표
- [ ] 뉴스-데이터 연관 분석
  - 정책 발표 vs 지표 변화 분석
  - 정책 효과 검증 리포트
